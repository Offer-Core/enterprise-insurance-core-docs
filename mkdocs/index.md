# Enterprise Insurance Platform

Welcome to the architecture and documentation hub for the Saudi Arabia insurance platform.

## Architecture Documentation

This site contains the core architecture documentation for the insurance platform.

- [Architecture Overview](knowledge-base/architecture/overview.md) — purpose, goals, principles, and high-level structure
- [Domain Architecture](knowledge-base/architecture/domain-architecture.md) — bounded contexts, core domains, and domain responsibilities
- [Integration Architecture](knowledge-base/architecture/integration-architecture.md) — APIs, events, and external system connections
- [API Architecture](knowledge-base/architecture/api-architecture.md) — API layers, gateway, communication patterns, and topology
- [Data Architecture](knowledge-base/architecture/data-architecture.md) — persistence model, schemas, and migration approach
- [UI Architecture](knowledge-base/architecture/ui-architecture.md) — dynamic forms, portal structure, and configurable experience
- [Operations and Observability](knowledge-base/architecture/operations-observability.md) — monitoring, alerts, and service health
- [Architecture Decisions](knowledge-base/architecture/architecture-decisions.md) — key ADRs and design rationale
- [Security Architecture](knowledge-base/architecture/security-architecture.md) — authentication, authorization, data protection, and controls
- [Deployment Architecture](knowledge-base/architecture/deployment-architecture.md) — environments, runtime topology, and observability

## Working Guidelines

- Keep each document focused on one architectural concern.
- Update the documents whenever a significant design decision changes.
- Prefer short, structured sections with clear ownership and status.
- Use links between documents to connect related decisions and components.
