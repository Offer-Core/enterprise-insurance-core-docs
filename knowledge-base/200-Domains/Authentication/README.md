# Authentication & Security Specification

This document details the authentication protocols, token management policies, Role-Based Access Control (RBAC) scopes, and Spring Security configurations for the Enterprise Insurance Platform.

---

## 1. Authentication Protocols & Flows

The platform enforces centralized token-based identity management using an OpenID Connect (OIDC) compliant Identity Provider (IdP) (such as Keycloak).

### 1.1 SPA Flow (Angular)
- **Flow Pattern:** OAuth2 Authorization Code Flow with **PKCE (Proof Key for Code Exchange)**.
- **Storage:** Access tokens are stored in-memory inside the Angular SPA state. Refresh tokens are stored as **HttpOnly, Secure, SameSite=Strict** cookies generated by the gateway proxy to mitigate Cross-Site Scripting (XSS) risks.
- **Client Library:** Angular OAuth2 OIDC library.

### 1.2 Service-to-Service Flow
- **Flow Pattern:** OAuth2 Client Credentials Grant.
- **Mechanism:** Internal microservices authenticate with the IdP using client credentials and cache access tokens, passing them via the HTTP `Authorization: Bearer <token>` header on downstream calls.

---

## 2. Token & Session Policies

- **Access Token Format:** JSON Web Token (JWT) signed using the RS256 algorithm.
- **Lifespan Rules:**
  - Access Token: **15 minutes** (short-lived).
  - Refresh Token: **8 hours** (sliding session).
- **JWT Mandatory Claims:**
  ```json
  {
    "iss": "https://auth.enterprise-insurance.sa/realms/insurance-platform",
    "sub": "usr_90a1b2c3-4d5e",
    "aud": "insurance-backend-api",
    "exp": 1783389000,
    "preferred_username": "k.saeed@agent.com",
    "roles": ["AGENT", "PARTNER"],
    "scope": "openid email profile policy:read policy:create"
  }
  ```

## 3. Role-Based Access Control (RBAC) Matrix

To secure operations, permissions are mapped to fine-grained scopes. Backend endpoints evaluate scopes, whereas the UI displays elements based on roles.

| Role | Description | Assigned Scopes |
| :--- | :--- | :--- |
| **CUSTOMER** | Policyholders | `policy:read`, `claims:read`, `claims:create` |
| **AGENT** | External brokers/agents | `policy:read`, `policy:create`, `claims:read`, `claims:create` |
| **UNDERWRITER**| Internal risk assessment staff | `policy:read`, `underwriting:evaluate`, `underwriting:override` |
| **CLAIMS_PROCESSOR**| Claims adjusters | `claims:read`, `claims:evaluate`, `claims:adjudicate` |
| **ADMINISTRATOR** | Platform configurations | `metadata:read`, `metadata:write`, `workflow:write` |

## 4. Spring Security Reference Implementation

All resource servers (backend domain APIs) must configure Spring Security to validate the OIDC tokens and enforce method-level security based on scopes.

### 4.1 Dependency Required
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-oauth2-resource-server</artifactId>
</dependency>
```

### 4.2 Security Configuration Class
```java
package com.enterprise.insurance.core.security;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.method.configuration.EnableMethodSecurity;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.oauth2.server.resource.authentication.JwtAuthenticationConverter;
import org.springframework.security.oauth2.server.resource.authentication.JwtGrantedAuthoritiesConverter;
import org.springframework.security.web.SecurityFilterChain;

@Configuration
@EnableWebSecurity
@EnableMethodSecurity // Enables @PreAuthorize on controllers
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable()) // Stateless APIs
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/actuator/health", "/actuator/info").permitAll()
                .requestMatchers("/v3/api-docs/**", "/swagger-ui/**").permitAll()
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt.jwtAuthenticationConverter(jwtAuthenticationConverter()))
            );
        return http.build();
    }

    @Bean
    public JwtAuthenticationConverter jwtAuthenticationConverter() {
        JwtGrantedAuthoritiesConverter grantedAuthoritiesConverter = new JwtGrantedAuthoritiesConverter();
        // Extract scopes from JWT claim "scope" and map as "SCOPE_xxx"
        grantedAuthoritiesConverter.setAuthorityPrefix("SCOPE_");
        grantedAuthoritiesConverter.setAuthoritiesClaimName("scope");

        JwtAuthenticationConverter jwtAuthenticationConverter = new JwtAuthenticationConverter();
        jwtAuthenticationConverter.setJwtGrantedAuthoritiesConverter(grantedAuthoritiesConverter);
        return jwtAuthenticationConverter;
    }
}
```
